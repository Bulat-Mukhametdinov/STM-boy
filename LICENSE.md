STM-boy uses a split-license model because hardware, firmware, and
documentation are reused in different ways.

## Hardware

Hardware design files, schematics, PCB layouts, Gerbers, BOM files,
manufacturing outputs, and future mechanical CAD files are licensed under:

`CERN-OHL-S-2.0`

This is the strongly reciprocal variant of the CERN Open Hardware Licence v2.
In practical terms, people can study, manufacture, and modify the hardware, but
published modified hardware designs should remain open under compatible terms.

## Firmware and Software

Firmware, scripts, and source code are licensed under:

`MIT`

This applies to files such as `hardware/breadboard/test_firmware.cpp` and future
source files under `firmware/` or `software/`.

## Documentation and Media

Documentation, README files, diagrams, images, videos, and demo media are
licensed under:

`CC-BY-4.0`

Attribution should name the project and author: STM-boy by Bulat Mukhametdinov.
