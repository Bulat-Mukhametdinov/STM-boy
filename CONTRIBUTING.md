# Contributing

STM-boy is currently an early hardware prototype. Contributions are most useful
when they keep the hardware, firmware, and documentation in sync.

## Hardware Changes

For PCB changes, include:

- Updated editable source design files.
- Updated schematic and PCB PDF exports.
- Updated BOM.
- Updated Gerbers if fabrication output changed.
- Notes about the affected hardware revision.
- Screenshots or renders when layout changes are visual.

## Firmware Changes

For firmware changes, include:

- Build instructions if new tools or libraries are required.
- Pinout updates when signal assignments change.
- A short test note describing the board or breadboard setup used.

## Documentation Changes

Keep public docs practical and builder-focused:

- State what is tested and what is planned.
- Prefer tables for pinouts, BOM notes, and measurements.
- Add warnings for battery, USB, power, or assembly risks.
